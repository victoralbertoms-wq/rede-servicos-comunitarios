Como colocar o projeto para rodar em qualquer máquina em poucos minutos

☁️ Como funciona

O código fica no GitHub. O banco de dados fica no Firebase. As imagens ficam no Cloudinary. Tudo na nuvem.

Para rodar em um computador novo, você só precisa clonar o repositório, criar o arquivo .env com as credenciais e instalar as dependências. Nenhum dado precisa ser transferido entre máquinas.

1️⃣ Instalar os Programas Necessários

A

Node.js 18 ou superior

nodejs.org — marque "Add to PATH" durante a instalação

Após instalar, abra o terminal e verifique:

node --version   # deve mostrar v18.x ou superior
npm --version
B

Git

git-scm.com — escolha "Git from the command line" durante a instalação

git --version
C

Firebase CLI

Instalar via npm após ter o Node.js instalado

npm install -g firebase-tools
firebase --version

2️⃣ Clonar o Repositório

Abra o terminal (PowerShell no Windows, Terminal no Mac/Linux) e rode:

git clone https://github.com/victoralbertoms-wq/rede-servicos-comunitarios.git
cd rede-servicos-comunitarios
💡 Isso baixa todo o código do GitHub para o seu computador. Demora alguns segundos.

3️⃣ Criar o Arquivo .env

⚠️ O arquivo .env não vai para o GitHub por segurança (está no .gitignore). Por isso precisa ser criado manualmente em cada computador novo.

Dentro da pasta web/, crie um arquivo chamado .env com este conteúdo:

VITE_FIREBASE_API_KEY=AIzaSyDzHsXE4jc8pvSwaDl171xVXbA_qqXBwf4
VITE_FIREBASE_AUTH_DOMAIN=rede-servicos.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=rede-servicos
VITE_FIREBASE_MESSAGING_SENDER_ID=1008910813540
VITE_FIREBASE_APP_ID=1:1008910813540:web:5b7ce8485fa4b095c25271
VITE_FIREBASE_MEASUREMENT_ID=
VITE_CLOUDINARY_CLOUD_NAME=dh3v2u54h
VITE_CLOUDINARY_UPLOAD_PRESET=rede-servicos
💡 No Windows: abra o Bloco de Notas, cole o conteúdo acima e salve como .env (com o ponto na frente) dentro da pasta web/. Certifique-se de que não ficou .env.txt.

4️⃣ Instalar Dependências e Rodar

cd web
npm install
npm run dev
✅ Abra o navegador em http://localhost:3000 — o sistema vai aparecer conectado ao mesmo banco de dados de produção.

O npm install pode demorar 1–2 minutos na primeira vez (baixa as dependências).

5️⃣ Firebase CLI (só para deploy)

Se precisar fazer deploy das regras ou do site a partir deste computador:

firebase login

Vai abrir o navegador para fazer login com a conta Google do projeto. Depois:

cd firebase
firebase deploy --only firestore:rules

🚀 Fazer Alterações e Publicar

O site em produção é atualizado automaticamente quando você faz git push. O GitHub Actions cuida do deploy.

# 1. Fazer as alterações no código

# 2. Salvar e publicar
git add .
git commit -m "descrição do que foi alterado"
git push
🌐 Em 1–2 minutos o site em https://rede-servicos.web.app estará atualizado.

⚠️ Erro de Credenciais do Git

No Windows, ao fazer git push pela primeira vez em um computador novo, pode aparecer erro de autenticação. Corrija assim:

git remote set-url origin https://victoralbertoms-wq:SEU_TOKEN@github.com/victoralbertoms-wq/rede-servicos-comunitarios.git

Como gerar o token do GitHub

  1. Acesse github.com → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Clique em "Generate new token (classic)"
  3. Marque os escopos: repo + workflow
  4. Copie o token gerado e substitua SEU_TOKEN no comando acima
🔒 Guarde o token em local seguro. Ele aparece apenas uma vez no GitHub.
